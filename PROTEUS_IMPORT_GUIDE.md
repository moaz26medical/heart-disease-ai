# 🔬 دليل استيراد الدائرة في Proteus

## 📋 نظرة عامة

هذا الدليل يشرح كيفية استيراد دائرة نظام تشخيص أمراض القلب في برنامج Proteus وتجهيزها للاختبار.

---

## 🚀 الخطوات السريعة

### 1️⃣ تحضير الملفات

```bash
# الملفات المطلوبة:
✅ proteus_circuit.xml          # ملف الدائرة
✅ arduino_ecg_sensor.ino       # كود الأردوينو
✅ PROTEUS_IMPORT_GUIDE.md      # هذا الدليل
```

### 2️⃣ فتح Proteus

1. افتح برنامج **Proteus Design Suite**
2. اختر **File → New Project**
3. أعطِ المشروع اسماً: `Heart_Disease_Diagnosis`

### 3️⃣ إنشاء الدائرة يدويًا

بما أن ملف XML يحتاج معالجة خاصة، سنقوم بإنشاء الدائرة خطوة بخطوة:

---

## 📐 خطوات بناء الدائرة

### المرحلة 1️⃣: إضافة المكونات الأساسية

#### الخطوة 1: إضافة Arduino Uno

```
1. اذهب إلى: Library → Search
2. ابحث عن: "ARDUINO_UNO"
3. اسحب المكون إلى الشاشة
4. ضعه في الموضع: X=100, Y=200
```

#### الخطوة 2: إضافة AD8232 ECG Sensor

```
1. اذهب إلى: Library → Search
2. ابحث عن: "AD8232"
3. اسحب المكون إلى الشاشة
4. ضعه في الموضع: X=400, Y=200
```

#### الخطوة 3: إضافة ADS1115 ADC

```
1. اذهب إلى: Library → Search
2. ابحث عن: "ADS1115"
3. اسحب المكون إلى الشاشة
4. ضعه في الموضع: X=300, Y=400
```

#### الخطوة 4: إضافة OLED Display

```
1. اذهب إلى: Library → Search
2. ابحث عن: "OLED_128x64"
3. اسحب المكون إلى الشاشة
4. ضعه في الموضع: X=600, Y=200
```

### المرحلة 3️⃣: إضافة المكونات الإضافية

### المرحلة 4️⃣: التوصيلات الكهربائية

#### توصيلات الطاقة

```
من Power Supply 5V:
1. اسحب سلك أحمر من (+) إلى Arduino 5V
2. اسحب سلك أسود من (-) إلى Arduino GND

من Power Supply 3.3V:
1. اسحب سلك أحمر من (+) إلى AD8232 3.3V
2. اسحب سلك أحمر من (+) إلى ADS1115 VDD
3. اسحب سلك أحمر من (+) إلى OLED VDD
4. اسحب أسلاك سوداء من (-) إلى جميع GND
```

#### توصيلات AD8232

```
AD8232 OUTPUT → Arduino A0 (سلك أصفر)
AD8232 LO+ → Arduino D2 (سلك برتقالي)
AD8232 LO- → Arduino D3 (سلك برتقالي)
AD8232 SDN → Arduino D4 (سلك بنفسجي)
```

#### توصيلات I2C

```
Arduino SDA → ADS1115 SDA (سلك أخضر)
Arduino SCL → ADS1115 SCL (سلك أزرق)
Arduino SDA → OLED SDA (سلك أخضر)
Arduino SCL → OLED SCL (سلك أزرق)
```

#### توصيلات LED

```
Arduino D5 → LED الأحمر (سلك أحمر)
Arduino D6 → LED الأخضر (سلك أخضر)
```

#### توصيلات الأقطاب

```
AD8232 RA → Electrode RA (سلك أحمر)
AD8232 LA → Electrode LA (سلك أصفر)
AD8232 LL → Electrode LL (سلك أسود)
```

## 🔧 تحميل الكود على Arduino

### الخطوة 1: فتح كود الأردوينو

```
1. في Proteus، انقر مزدوجاً على Arduino Uno
2. اختر: "Edit Properties"
3. اختر: "Program File"
4. استعرض واختر: "arduino_ecg_sensor.ino"
```

### الخطوة 2: تجميع الكود

```
1. اذهب إلى: Tools → Compile
2. تأكد من عدم وجود أخطاء
3. إذا كان هناك أخطاء، تحقق من الملف
```

## 🧪 الاختبار

### اختبار 1️⃣: اختبار الطاقة

```
1. اضغط على زر التشغيل (Play)
2. تحقق من:
   ✅ جهد 5V على Arduino
   ✅ جهد 3.3V على AD8232
   ✅ جهد 3.3V على ADS1115
   ✅ جهد 3.3V على OLED
```

### اختبار 2️⃣: اختبار الاتصال

```
1. افتح: Tools → Virtual Instruments
2. اختر: Serial Monitor
3. تحقق من رسائل الاتصال
4. يجب أن ترى: "System initialized"
```

### اختبار 3️⃣: اختبار الإشارة

```
1. في Proteus، اختر: Tools → Signal Generator
2. أنشئ إشارة جيبية:
   - التردد: 1 Hz
   - السعة: 1V
3. وصلها إلى AD8232 OUTPUT
4. تحقق من قراءة الإشارة
```

### اختبار 4️⃣: اختبار LED

```
1. تحقق من إضاءة LED الأخضر (عمل طبيعي)
2. قم بفصل أحد الأقطاب
3. تحقق من إضاءة LED الأحمر (خطأ)
```

### اختبار 5️⃣: اختبار OLED

```
1. تحقق من ظهور النصوص على الشاشة
2. يجب أن ترى:
   ✅ "Heart Disease Diagnosis"
   ✅ معدل ضربات القلب
   ✅ نتيجة التنبؤ
```

## ⚠️ حل المشاكل الشائعة

### المشكلة 1️⃣: لا يظهر أي شيء على الشاشة

```
الحل:
1. تحقق من توصيل I2C (SDA و SCL)
2. تحقق من عنوان OLED (0x3C)
3. تأكد من توصيل الطاقة 3.3V
4. جرب إعادة تشغيل المحاكاة
```

### المشكلة 2️⃣: لا تظهر إشارة ECG

```
الحل:
1. تحقق من توصيل AD8232 OUTPUT إلى Arduino A0
2. تحقق من توصيل الأقطاب الثلاثة
3. تأكد من تشغيل Signal Generator
4. تحقق من قيمة الإشارة (يجب أن تكون بين 0-5V)
```

### المشكلة 3️⃣: LED الأحمر يضيء دائماً

```
الحل:
1. تحقق من توصيل الأقطاب
2. تأكد من أن جميع الأقطاب موصولة
3. تحقق من كود الأردوينو
4. جرب إعادة تشغيل المحاكاة
```

### المشكلة 4️⃣: أخطاء I2C

```
الحل:
1. تحقق من المقاومات Pull-up (R1, R2)
2. تأكد من توصيل SDA و SCL بشكل صحيح
3. تحقق من عناوين I2C (ADS1115: 0x48, OLED: 0x3C)
4. استخدم I2C Scanner للتحقق من الأجهزة
```
